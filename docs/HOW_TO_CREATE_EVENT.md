# 이벤트 페이지 작업 가이드

이 문서는 기획자가 직접 이벤트 페이지를 만들 때 참고하는 가장 쉬운 작업 안내서입니다.

## 📌 이벤트 페이지 만드는 방법

1. `events/sample-event` 폴더를 복사합니다.
2. 복사한 폴더 이름을 원하는 이벤트 이름으로 변경합니다. 예: `event-001`
3. 폴더 안의 이미지 파일을 새 이미지로 교체합니다.
4. 같은 폴더 안의 `index.html` 파일을 열어 문구와 링크를 수정합니다.

예시 구조:

```text
events/
  event-001/
    index.html
    banner.webp
    detail-01.webp
    detail-02.webp
```

## 📌 이미지 규칙

- 메인 배너 이미지는 `banner.webp` 파일명을 사용합니다.
- 상세 이미지는 `detail-01.webp`, `detail-02.webp` 파일명을 사용합니다.
- 이미지 형식은 `webp`를 권장합니다.
- 이미지 비율은 모바일 화면에서 잘 보이도록 세로형 또는 직사각형 구성을 추천합니다.

## 📌 HTML 수정 방법

수정이 필요한 부분은 아래 3가지가 가장 중요합니다.

- 제목 수정: 브라우저 제목과 페이지 메인 제목을 바꿉니다.
- 설명 수정: 이벤트 소개 문구와 각 섹션 설명을 바꿉니다.
- 버튼 링크 수정: 참가 신청 또는 상세 페이지 링크를 바꿉니다.

수정 예시:

```html
<title>봄맞이 미플온 이벤트</title>
<h1>봄맞이 미플온 이벤트</h1>
<p>신규 보드게임 체험과 현장 할인 혜택을 만나보세요.</p>
<a class="cta-button" href="https://example.com">지금 신청하기</a>
```

## 📌 주의사항

- 파일명은 반드시 유지하세요.
- 이미지 경로는 상대경로를 유지하세요. 예: `./banner.webp`
- 외부 JS 추가 금지입니다.
- HTML을 수정할 때 기존 구조를 너무 크게 바꾸지 않으면 오류를 줄일 수 있습니다.

## 📌 배포 방법

- 작업 후 저장소에 `git push` 하면 GitHub Pages에 자동 반영됩니다.
- 반영까지 수 분 정도 걸릴 수 있습니다.
- 배포 후에는 실제 URL에서 이미지와 버튼 링크가 정상인지 꼭 확인하세요.

## 추천 작업 순서

1. `sample-event` 복사
2. 폴더 이름 변경
3. 이미지 교체
4. `index.html` 제목과 설명 수정
5. 버튼 링크 수정
6. 저장 후 커밋 및 `git push`
